# Product design

This directory contains the accepted visual concepts and extracted design system for the deployable application. Frontend work treats these files as fidelity references; runtime agents do not use them as company memory.

- `chat-workspace-concept.png` defines the primary chat surface.
- `automation-builder-concept.png` defines the workflow-authoring surface.
- `DESIGN-SYSTEM.md` records the implementation tokens and component inventory extracted from both concepts.
